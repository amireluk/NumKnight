1

Start the dev server

Run once per session from the project root:

npm run dev

Then open http://localhost:5173/NumKnight/animation-test.html in your browser.

2

Edit this file, refresh to see changes

This file is a standalone HTML in public/ — Vite serves it directly. Edit public/animation-test.html and hit F5 to refresh. No build step, no hot-reload lag.

3

Tune timing with the Settings tab

Sliders update live. When you find values you like, note them down — then copy the numbers into the Framer Motion transition objects in the game components.

4

Port animations into React components

The game uses Framer Motion for all character animations. The CSS animations here map 1:1 to Framer Motion's API:

// CSS: translateX(0 → 54px → 0) over 0.45s
// Framer Motion equivalent:
animate={{ x: [0, 54, 0] }}
transition={{ duration: 0.45, times: [0, 0.4, 1], ease: 'easeInOut' }}

Key files to edit: src/components/KnightCharacter.jsx, src/components/EnemyCharacter.jsx

5

Test in the real game

Dev server is already running — switch to http://localhost:5173/NumKnight/ and play through a battle. Use Easy mode (1 battle/world) for the fastest loop.

6

Deploy to dev when ready

npm run deploy:dev

Then commit and push following the sequence in CLAUDE.md. This file (animation-test.html) deploys with the rest of public/ — that's fine for now.

Sprite naming convention:
assets/characters/{enemy-id}/{enemy-id}-{pose}.webp
Poses: idle · attack · hit · dead — Knight also has camp · victory
